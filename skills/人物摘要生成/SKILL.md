# 人物摘要生成

按姓名分组聚合案件中每个人的全部笔录，生成综合人物摘要，以检察官/律师阅卷视角突出对定罪量刑有意义的信息。

## 触发关键词

- `"生成人物摘要"`、`"人物摘要"`、`"人物画像"`
- `"人物分析"`、`"角色摘要"`、`"嫌疑人摘要"`
- `"综合摘要"`、`"人物梳理"`、`"人物关系"`

## 前置条件

- 案件JSON已存在于 `<项目根目录>\project\data\{案件名}.json`
- 建议先完成 `/笔录摘要生成`，利用已有的"笔录摘要"字段提高效率
- 如有起诉书内容，优先结合分析
- **必须先确认委托当事人与涉案核心人物，再做人像摘要**

## 核心原则

### 先定委托当事人，再定角色

人物摘要不是“所有出现过的人的流水账”，而是围绕委托当事人和与其有证明关系的人物来展开。

- 委托当事人是人物摘要判断优先级的第一依据
- 非委托当事人也可以生成摘要，但必须说明其与委托当事人、哪一事实链条相关
- 如果某人与委托当事人及案件主线关联弱，应降低摘要权重，避免喧宾夺主

## 数据格式

人物摘要存储在案件JSON的顶层字段 `人物摘要` 中，以人名为键。每人包含丰富的结构化字段：

```json
{
  "案件名称": "...",
  "人物摘要": {
    "张三": {
      "姓名": "张三",
      "角色定位": "主犯",
      "笔录数量": 19,
      "总字数": 41890,
      "认罪态度": "认罪认罚",
      "综合摘要": "【角色定位】张三在本案中被认定为主犯。..."
    },
    "孙八": {
      "姓名": "孙八",
      "角色定位": "证人",
      "笔录数量": 1,
      "总字数": 4696,
      "认罪态度": "配合调查",
      "综合摘要": "..."
    }
  }
}
```

**前端适配：** 前端会读取 `综合摘要`（或兼容 `content`）字段作为显示内容，`笔录数量`（或兼容 `record_count`）作为计数，`角色定位` 和 `认罪态度` 显示为标签。

## 工作流程

### 步骤1：确认案件名

询问用户要处理的案件名称。

### 步骤2：读取并分组

读取案件JSON，按姓名分组聚合笔录。

### 步骤2.5：读取起诉意见书并锁定委托当事人

所有子agent生成摘要前，必须共享以下上下文：
- 起诉意见书/起诉书简要版
- 委托当事人
- 涉案核心人物

角色定位优先依据：
1. 起诉意见书明确记载
2. 是否属于委托当事人
3. 笔录中与委托当事人的关系与行为

### 步骤3：逐人生成综合摘要

**重要：必须使用多agent并行处理。** 人物数量通常超过20人，单线程处理极慢。

**策略：**
- 人物数量 ≤ 10：逐人生成
- 人物数量 > 10：使用 executor agent 并行处理，每个agent处理一批人物
- 启动agent时使用 `mode: bypassPermissions` 以避免文件读写权限被拒

**Prompt模板（每个人物）：**

```
请为以下案件人物生成综合摘要，控制在500字以内，以检察官/律师阅卷视角撰写：

【案件名称】{案件名称}
【案件主线】{起诉意见书简要版}
【委托当事人】{entrusted_parties}
【涉案核心人物】{related_people}
【人物姓名】{姓名}
【该人物笔录列表】
{按时间排序的笔录摘要列表，每份包含：日期、笔录类型、笔录摘要/内容摘要}

【综合摘要要求】
1. 人物角色定位：该人在案件中的角色（主犯/从犯/证人/被害人等）
2. 时间线梳理：该人物相关的关键事件按时间排序
3. 核心供述要点：各次笔录中的核心供述内容
4. 与其他人的关系网络：该人与案件中其他关键人物的关系
5. 矛盾点/翻供点：如有前后矛盾或翻供，明确指出

要求：
- 先判断该人物是否属于委托当事人；如果不是，明确其与委托当事人、哪条事实链条相关
- 必须说明该人物对委托当事人的辩护价值或风险点
- 以检察官/律师阅卷视角撰写
- 突出对定罪量刑有意义的信息
- 控制在500字以内
- 使用第三人称客观叙述
- 输出格式为纯文本段落，不要分点编号
```

**输入数据准备：**

对每份笔录，优先使用以下字段（按优先级）：
1. "笔录摘要"（如已生成）
2. "内容摘要"（导入时生成的前200字）
3. "全文内容"前500字（fallback）

### 步骤4：写入JSON

将生成的人物摘要写入案件JSON的顶层 `人物摘要` 字段，保存回文件。

```python
def save_person_summaries(case_data, person_summaries, json_path):
    """将人物摘要写入案件JSON并保存。

    Args:
        case_data: 案件JSON数据
        person_summaries: {姓名: {"姓名", "角色定位", "笔录数量", "总字数", "认罪态度", "综合摘要"}}
        json_path: JSON文件路径
    """
    case_data['人物摘要'] = person_summaries

    with open(json_path, 'w', encoding='utf-8') as f:
        json.dump(case_data, f, ensure_ascii=False, indent=2)

    return json_path
```

### 步骤5：汇报结果

向用户汇报：
- 案件中涉及人物总数
- 成功生成人物摘要数
- 人物角色概览（简要列表）
- 示例人物摘要1-2条

## 摘要质量标准

| 维度 | 要求 |
|------|------|
| 字数 | ≤ 500字 |
| 视角 | 检察官/律师阅卷视角 |
| 重点 | 对定罪量刑有意义的信息 |
| 人称 | 第三人称客观叙述 |
| 完整性 | 覆盖角色、时间线、供述、关系、矛盾点 |
